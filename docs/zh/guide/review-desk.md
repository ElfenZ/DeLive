# AI 复盘工作台

复盘工作台是用于探索已完成转录会话的全页工作区。从历史记录视图打开，提供六个标签页和动画滑动标签栏：**转录**、**AI 纠错**、**概览**、**AI 分析**、**对话**、**思维导图**。

## 概览标签页

使用 OpenAI 兼容的 chat completions 端点从转录生成 AI 简报。

**输出包括：**
- **摘要** — 会话的简洁概述
- **行动项** — 提取的任务和后续步骤
- **关键词** — 讨论中的关键术语
- **章节** — 带可选时间戳的逻辑分段
- **标题建议** — AI 生成的会话标题
- **标签建议** — 用于组织的相关标签

一键 **应用** 按钮可直接采纳建议的标题和标签。

![AI 概览](/images/screenshot-ai-overview.png)

### 配置

AI 功能需要在 **设置 > 通用 > AI 后处理** 中配置 OpenAI 兼容端点：

- **Base URL** — 默认 `http://127.0.0.1:11434/v1`（Ollama）
- **模型** — 用于生成的模型
- **API Key** — 可选，取决于端点
- **提示语言** — `zh`（中文）或 `en`（英文）

## 转录标签页

显示完整转录：

- 左侧沟栏中的时间戳分段
- 彩色说话人标记（支持说话人分离时）
- 连续同说话人合并
- 单个分段悬停高亮
- 可选翻译文本块

**导出格式：** TXT、Markdown、SRT、VTT

![转录详情](/images/screenshot-transcript-detail.png)

## AI 纠错标签页

提供两种纠错模式：

- **直接纠错** — 分片检测局部 ASR 修改意图，通过本地硬校验的 Patch 会自动应用
- **先检测后纠错** — 分片检测合法候选，默认全部选中；可用“全部选中 / 全部不选”调整，确认后只在本地应用，不会再次调用模型

原始转录始终不变。纠错结果由原文和已应用 Patch 确定性生成，提供真实 Diff、单条撤销/恢复和全部恢复原文。纠错结果和修正稿 TXT/Markdown 会保留原说话人标签与时间；跨说话人边界或边界插入等无法可靠归属的新增内容会单独标为 `S? / 说话人不确定`，不会伪造归属。若历史会话中的原分段已经无法与完整原文对应，页面和导出会先保留原说话人分段，再附“无法安全分段的完整修正稿”。长文本按非重叠可编辑区分片处理，只有全部分片成功后才发布新结果；失败或暂停不会覆盖上一版结果。SRT/VTT 继续使用原始 ASR 数据。

### 智能文本源选择

纠错完成后，后续 AI 功能（概览、对话、思维导图、AI 分析）自动使用纠错后的文本。偏好可在 **设置 > AI** 中配置：

| 选项 | 行为 |
|------|------|
| **自动** *（默认）* | 有纠错文本时使用纠错版本，否则使用原始文本 |
| **始终原始** | 始终使用原始转录文本 |
| **始终纠错后** | 有已发布纠错结果时使用该结果，否则回退原始文本 |

各标签页顶部实时显示横幅，提示当前使用的文本源。

## 对话标签页

关于转录的多线程 AI 对话：

- GFM Markdown 渲染，带语法高亮代码块
- 一键复制代码
- 用户/AI 头像
- 悬停操作（复制、重新生成）
- 动画思考点指示器
- 自动调整大小的文本输入框（Enter 发送）
- 悬浮滚动到底部按钮
- 按线程删除

每个问题发送时附带转录上下文和最多 4 轮先前问答以保持对话连续性。回答包含可选的 `citations`，引用转录中的特定片段。

![AI 对话](/images/screenshot-ai-chat.png)

## 思维导图标签页

从会话生成 Markmap 兼容的 Markdown 思维导图：

- 实时 Markdown 编辑器
- 交互式 Markmap 可视化
- 全屏 portal 模式
- 导出为 SVG 或 PNG

思维导图提示在可用时会利用 AI 摘要、行动项和关键词。

![思维导图](/images/screenshot-mindmap.png)

## 键盘导航

| 快捷键 | 动作 |
|--------|------|
| `←` / `→` | 切换标签页 |
| `Ctrl/Cmd + 1–6` | 跳转到特定标签页 |
| `Escape` | 关闭复盘工作台 |
